# InvestmentEngine

Automated investment research and scoring platform for public companies.

## Current scope

- Source of truth: `Ticker-Master.json`
- Main project folder: `investment-engine`
- Sector parent directory: `investment-engine/sector`
- Data source: Finviz Elite (CSV export API)
- Output artifacts:
  - `investment-engine/sector/<SECTOR>/stock-data/investment_data_MMDDYYYY.xlsx` — weekly scored workbook (generated by GitHub Actions)
- Automation: GitHub Actions scheduled and manual runs

> **Note:** Claude ticker review is currently disabled. The `ticker-review/` output and `ANTHROPIC_API_KEY` secret are not active. The logic remains in `data_engine.py` (commented out) and can be re-enabled when ready.

## Workbook model

Each workbook contains the following tabs in order:

| Tab | Description |
|---|---|
| **Investment Master** | All tickers across all pillars, sorted by Investment Score descending |
| **Audit** | Raw Finviz values and intermediate calculations for every ticker — use this to verify any computed column |
| **Formula Guide** | Documents every calculated column: the formula, how to verify it, and the Investment Score weight breakdown |
| **AI [Sub-pillar]** | One tab per pillar with full scored data for that pillar (follows the first three fixed tabs) |

### Formula Guide tab

Documents each calculated column with its formula, step-by-step verification instructions, and data source notes. Also includes the full Investment Score weight and normalization table.

## Investment Score

A weighted 0–100 ranking designed to identify the most attractive investment opportunities. Metrics are normalized to a 0–100 component score, then combined using the weights below. Missing metrics are excluded and the remaining weights are rescaled proportionally so partial data still produces a meaningful score.

### Weights

| Category | Metric | Weight |
|---|---|---|
| **Growth (30%)** | Revenue Growth % | 20% |
| | EPS Growth % | 10% |
| **Financial Quality (25%)** | Gross Margin % | 10% |
| | Net Profit Margin % | 10% |
| | Debt/Equity | 5% |
| **Valuation (20%)** | P/S Ratio | 10% |
| | Upside % | 10% |
| | P/E Ratio | 5% |
| **Entry Timing (20%)** | RSI | 10% |
| | Buy Zone Proximity | 5% |
| | Analyst Recom | 5% |

**Scoring direction:**
- Higher is better: Revenue Growth, EPS Growth, Gross Margin, Net Profit Margin, Upside %
- Lower is better: P/E Ratio (capped at 60; negative = 0), P/S Ratio (capped at 30), Debt/Equity (capped at 3.0), Analyst Recom (1=Strong Buy scores 100, 5=Strong Sell scores 0)
- RSI: scored highest at 30 (oversold), linearly to 0 at 80 (overbought)
- Buy Zone Proximity: scored 100 at or below buy zone, 0 when 25%+ above it

## Target Price methodology

Target Price is the analyst consensus price target sourced directly from Finviz. Upside % is calculated as:

```
Upside % = ((Target Price − Current Price) / Current Price) × 100
```

## Graham Number

```
Graham Number = √(22.5 × EPS (ttm) × Book Value Per Share)
```

Both EPS (ttm) and Book Value Per Share (Book/sh) are sourced from Finviz.

`Graham Undervalued` is an informational boolean column only — it does not feed into the Investment Score. It is set to `None` (not `False`) when EPS or Book Value Per Share data is unavailable or non-positive.

The Graham Number intermediate value is visible in the Audit tab for verification.

## Finviz data source

The engine calls the Finviz Elite export endpoint once per ticker:

```
GET https://elite.finviz.com/export?v=151&t={TICKER}&c={COLUMNS}&auth={TOKEN}
```

Returns a CSV with one data row per ticker. Column codes used:

| Code | Field |
|---|---|
| 1 | Ticker |
| 65 | Price |
| 59 | RSI (14) |
| 7 | P/E |
| 10 | P/S |
| 6 | Market Cap |
| 69 | Target Price |
| 16 | EPS (ttm) |
| 22 | EPS Growth Q/Q |
| 19 | EPS Growth Past 5Y |
| 73 | Book/sh |
| 23 | Sales Growth Q/Q |
| 21 | Sales Growth Past 5Y |
| 12 | P/Cash |
| 38 | Total Debt/Equity |
| 41 | Profit Margin |
| 39 | Gross Margin |
| 48 | Beta |
| 26 | Insider Ownership |
| 28 | Institutional Ownership |
| 62 | Analyst Recom |

## Local setup

1. Create a virtual environment and install dependencies.
2. Set `FINVIZ` in your environment (your Finviz Elite auth token from the export URL).
3. Run the engine.

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

$env:FINVIZ = "your_finviz_auth_token_here"
python investment-engine/engine/data_engine.py
```

## Configure tickers

Edit `investment-engine/Ticker-Master.json` to control the full multi-sector ticker universe. Each sector file `investment-engine/sector/<SECTOR>/pillar_tickers.json` is kept in sync automatically on every run.

To add a new pillar, add it directly to `Ticker-Master.json` under the appropriate sector key. The engine will create the required directories on its next run.

## GitHub Actions automation

Workflow file: `.github/workflows/data_engine.yml`

Triggers:
- Manual: `workflow_dispatch`
- Schedule: Weekly on Sundays at 21:00 UTC (4:00 PM EST)

Required repository secrets:

| Secret | Purpose |
|---|---|
| `FINVIZ` | Finviz Elite auth token (from the `auth=` parameter in your export URL) |

Add secrets in GitHub: **Settings → Secrets and variables → Actions → New repository secret**

On each run the workflow:
1. Fetches market data from Finviz and scores all tickers
2. Writes `investment_data_MMDDYYYY.xlsx` per sector with all scored columns plus Audit and Formula Guide tabs
3. Commits and pushes any changed files
