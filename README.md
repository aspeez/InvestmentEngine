## Investment Score

A weighted 0â€“100 ranking designed to identify the most attractive investment opportunities. Metrics are normalized to a 0â€“100 component score, then combined using the weights below. Missing metrics are excluded and the remaining weights are rescaled proportionally so partial data still produces a meaningful score.

### Weights

| Category | Metric | Weight |
|---|---|---|
| **Growth (35%)** | Revenue Growth % | 20% |
| | EPS Growth % | 10% |
| | Backlog Growth % | 5% |
| **Financial Quality (25%)** | Gross Margin % | 15% |
| | Net Cash Ratio | 10% |
| **Valuation (25%)** | P/E Ratio | 5% |
| | P/S Ratio | 10% |
| | Upside % | 10% |
| **Entry Timing (15%)** | RSI | 10% |
| | Buy Zone Proximity | 5% |

**Scoring direction:**
- Higher is better: Revenue Growth, EPS Growth, Backlog Growth, Gross Margin, Net Cash Ratio, Upside %
- Lower is better: P/E Ratio (capped at 60; negative = 0), P/S Ratio (capped at 30)
- RSI: scored highest at 30 (oversold), linearly to 0 at 80 (overbought)
- Buy Zone Proximity: scored 100 at or below buy zone, 0 when 25%+ above it

## Target Price methodology

```
Target Price = EPS Ã— Pillar P/E Multiple
```

**EPS source priority:**
1. Forward EPS â€” FMP `analyst-estimates` (annual, field `epsAvg`)
2. Earnings estimate â€” FMP `earnings` (field `epsEstimated`, future quarters only)
3. TTM EPS â€” FMP `ratios-ttm-bulk` (field `epsTTM`)

If all three are unavailable or zero, `Target Price` is set to `None` and a warning is printed:
```
[WARN] Target Price unavailable for {TICKER} â€” missing EPS data
```

## Graham Number

```
Graham Number = âˆš(22.5 Ã— EPS Ã— Book Value Per Share)
```

Book Value Per Share is sourced from FMP `ratios` (field `bookValuePerShare`, most recent annual period).

`Graham Undervalued` is an informational boolean column only â€” it does not feed into the Investment Score. It is set to `None` (not `False`) when EPS or Book Value Per Share data is unavailable or non-positive.

## FMP endpoints used

| Endpoint | Data pulled |
|---|---|
| `batch-quote` | Current Price |
| `technical-indicators/rsi` | RSI |
| `market-capitalization-batch` | Market Cap |
| `ratios-ttm-bulk` | P/E, P/S, Gross Margin, TTM EPS |
| `financial-growth` | Revenue Growth, EPS Growth |

| `earnings` | Forward EPS fallback (`epsEstimated`) |
| `ratios` | Book Value Per Share |

## Local setup

1. Create a virtual environment and install dependencies.
2. Set `FMP_API_KEY` (or `FMP`) in your environment.
3. Run the engine.

Get your FMP API key at: https://financialmodelingprep.com/

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

$env:FMP_API_KEY = "your_fmp_key_here"
python investment-engine/engine/data_engine.py
```

## Configure tickers

Edit `investment-engine/Ticker-Master.json` to control the full multi-sector ticker universe. Each sector file `investment-engine/sector/<SECTOR>/pillar_tickers.json` is kept in sync automatically on every run.

To add a new pillar, add it directly to `Ticker-Master.json` under the appropriate sector key. The engine will create the required directories on its next run.

## GitHub Actions automation

Workflow file: `.github/workflows/update_ai_workbook.yml`

Triggers:
- Manual: `workflow_dispatch`
- Schedule: Weekly on Sundays at 21:00 UTC (4:00 PM EST)

Required repository secrets:

| Secret | Purpose |
|---|---|
| `FMP` | FMP API key for market data |

Add secrets in GitHub: **Settings â†’ Secrets and variables â†’ Actions â†’ New repository secret**

On each run the workflow:
1. Fetches market data and scores all tickers
2. Writes `investment_data_MMDDYYYY.xlsx` per sector with all scored columns
3. Commits and pushes any changed files
